# 自動テストの追加

## 背景・目的

現在ユニットテストがゼロであり、VirtualHost 設定生成やスラグ検証などの重要なロジックが自動テストされていない。
不正な VirtualHost 定義を生成すると Apache が起動不能になるため、テストによる品質担保が必要。

## 作業内容

### テスト環境のセットアップ

- PHPUnit のインストール（composer require --dev）
- `phpunit.xml` の作成
- `tests/` ディレクトリの作成

### テスト対象と優先度

1. **VirtualHost 設定生成**（最重要）
   - HTTP VirtualHost の正しいフォーマット
   - HTTPS VirtualHost の正しいフォーマット
   - ディレクトリ型・プロキシ型・WebSocket 型それぞれの出力
   - 複数ベースドメインへの展開
   - スナップショットテストで出力の退行を検知

2. **slug バリデーション**
   - 有効なスラグの受理（`my-app`, `app1`）
   - 無効なスラグの拒否（先頭/末尾ハイフン、大文字、特殊文字）
   - インジェクション文字列の拒否

3. **ルート競合検出**
   - 同一 slug の shadowing 判定
   - グループ優先度に基づく解決順序
   - 明示ルートのグループルートに対する優先

4. **グループディレクトリスキャン**
   - 正常なディレクトリ構造のスキャン
   - `public/` 自動検出
   - 非スラグ名ディレクトリの除外

5. **データ永続化**
   - loadState のフォールバック動作（正常・破損・バックアップ復元）
   - atomicWrite の一貫性

## 完了条件

- PHPUnit が `composer test` で実行可能
- VirtualHost 生成のスナップショットテストが存在する
- slug バリデーションのテストカバレッジが100%
- 全テストがパスする
